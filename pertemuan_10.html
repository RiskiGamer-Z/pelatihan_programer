Fase 2: Backend Logic

Pertemuan 10: Penyempurnaan Pendataan Siswa Sederhana

Melanjutkan project pertemuan 9 hingga siap dijadikan output ujian.

A. Tujuan Pembelajaran

  • Menyempurnakan project pendataan siswa dari Pertemuan 9.
  • Menambahkan fitur hapus data per baris.
  • Menambahkan validasi input yang lebih baik.
  • Merapikan tampilan agar siap untuk Output 2.

B. Scope Penyempurnaan Hari Ini

  1. Tambah fitur hapus data.
  2. Tambah validasi lebih ketat (nama minimal 3 karakter).
  3. Tambah pesan sukses/gagal saat submit.
  4. Rapikan tampilan tabel agar mudah dibaca.

C. Alur Penyempurnaan Project

  • Data ditambah dari form, kemudian disimpan ke session.
  • Setiap baris data diberi tombol hapus.
  • Jika tombol hapus ditekan, data di indeks terkait dihapus.
  • Tabel diperbarui setelah tambah/hapus data.

D. Konsep Inti yang Dipakai

1. Hapus data per indeks array session

Setiap data di session punya indeks. Kita kirim indeks itu melalui form untuk menghapus baris tertentu.

<?php
if (isset($_POST['hapus'])) {
    $index = (int)($_POST['index'] ?? -1);
    if (isset($_SESSION['data_siswa'][$index])) {
        unset($_SESSION['data_siswa'][$index]);
        $_SESSION['data_siswa'] = array_values($_SESSION['data_siswa']);
    }
}
?>

Penjelasan kode:

  • isset($_POST['hapus']) mengecek apakah tombol hapus ditekan.
  • $index = (int)($_POST['index'] ?? -1) mengambil posisi data yang akan dihapus.
  • isset($_SESSION['data_siswa'][$index]) memastikan data pada indeks itu benar-benar ada.
  • unset(...) menghapus data pada indeks tersebut dari array session.
  • array_values(...) merapikan ulang index array agar urut dari 0 lagi.

2. Validasi sederhana sebelum simpan

<?php
$nama = trim($_POST['nama'] ?? '');
if (strlen($nama) < 3) {
    $pesanError = 'Nama minimal 3 karakter.';
}
?>

Penjelasan kode:

  • trim() menghapus spasi di depan dan belakang input.
  • $_POST['nama'] ?? '' memberi nilai kosong jika field belum ada.
  • strlen($nama) < 3 memeriksa apakah nama terlalu pendek.
  • $pesanError dipakai untuk menampilkan pesan kesalahan ke user.

3. Tabel + tombol aksi di setiap baris

<?php foreach ($_SESSION['data_siswa'] as $index => $siswa): ?>
<tr>
    <td><?php echo $index + 1; ?></td>
    <td><?php echo $siswa['nama']; ?></td>
    <td>
        <form method="POST" action="">
            <input type="hidden" name="index" value="<?php echo $index; ?>">
            <button type="submit" name="hapus">Hapus</button>
        </form>
    </td>
</tr>
<?php endforeach; ?>

Penjelasan kode:

  • foreach membaca semua data siswa satu per satu.
  • $index + 1 menampilkan nomor urut mulai dari 1.
  • <form method="POST"> dipakai agar setiap baris bisa punya tombol hapus sendiri.
  • <input type="hidden" name="index" ...> menyimpan posisi data yang akan dihapus.
  • Tombol Hapus mengirim request ke server untuk menghapus data itu saja.

E. Contoh Implementasi Lengkap

Contoh Full File: Tambah + Hapus + Validasi

<?php
session_start();

if (!isset($_SESSION['data_siswa'])) {
    $_SESSION['data_siswa'] = [];
}

$pesan = '';

if (isset($_POST['simpan'])) {
    $nama = trim($_POST['nama'] ?? '');
    $kelas = trim($_POST['kelas'] ?? '');
    $jurusan = trim($_POST['jurusan'] ?? '');

    if (strlen($nama) < 3 || $kelas === '' || $jurusan === '') {
        $pesan = 'Data gagal disimpan. Cek kembali input Anda.';
    } else {
        $_SESSION['data_siswa'][] = [
            'nama' => htmlspecialchars($nama),
            'kelas' => htmlspecialchars($kelas),
            'jurusan' => htmlspecialchars($jurusan)
        ];
        $pesan = 'Data berhasil ditambahkan.';
    }
}

if (isset($_POST['hapus'])) {
    $index = (int)($_POST['index'] ?? -1);
    if (isset($_SESSION['data_siswa'][$index])) {
        unset($_SESSION['data_siswa'][$index]);
        $_SESSION['data_siswa'] = array_values($_SESSION['data_siswa']);
        $pesan = 'Data berhasil dihapus.';
    }
}
?>

  • Validasi memastikan nama, kelas, dan jurusan terisi sebelum disimpan.
  • Bagian hapus menghapus data berdasarkan index lalu merapikan ulang array.
  • Bagian HTML menampilkan form input dan tabel data pada satu halaman.

I. Memecah PHP, CSS, dan JS dalam Satu Folder

Agar project lebih rapi, file PHP, CSS, dan JavaScript sebaiknya dipisahkan tetapi tetap berada di satu folder project yang sama.

Contoh struktur folder:

pendataan_siswa/
├── index.php
├── style.css
└── script.js

Penjelasan struktur:

  • index.php berisi logika PHP dan tampilan HTML utama.
  • style.css berisi semua pengaturan tampilan seperti warna, jarak, dan layout.
  • script.js berisi JavaScript kecil seperti konfirmasi hapus atau validasi ringan.

Cara menghubungkan file:

<link rel="stylesheet" href="style.css">
<script src="script.js"></script>

Kenapa dipisah?

  • Lebih mudah dibaca dan dirawat.
  • Kalau ada perubahan tampilan, cukup edit file CSS.
  • Kalau ada perubahan interaksi, cukup edit file JS.
  • File PHP jadi fokus ke proses data dan logic.

Untuk project pendataan siswa, pola ini sangat membantu saat project mulai membesar menjelang Output 2.
