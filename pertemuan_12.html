Pertemuan 12: Penyempurnaan CRUD (Header + CSS + JavaScript)

Di sesi ini siswa merapikan project CRUD: menambah header aplikasi, memperbaiki tampilan, dan memberi interaksi JavaScript agar aplikasi lebih siap demo.

A. Target Pertemuan 12

  • CRUD tetap berjalan penuh: tambah, tampil, edit, hapus.
  • Halaman memiliki header yang konsisten di semua file.
  • Tampilan dipisah ke style.css agar rapi dan mudah dikembangkan.
  • JavaScript dipakai untuk validasi ringan, konfirmasi aksi, dan pencarian cepat.
  • Struktur file project menjadi lebih profesional.

E. JavaScript Ringan untuk UX CRUD

// script.js
// Konfirmasi hapus
function konfirmasiHapus(nama) {
    return confirm('Yakin ingin menghapus data ' + nama + '?');
}

// Validasi form sederhana
function validasiFormSiswa() {
    const nama = document.getElementById('nama').value.trim();
    const kelas = document.getElementById('kelas').value.trim();
    const jurusan = document.getElementById('jurusan').value.trim();
    const email = document.getElementById('email').value.trim();

    if (nama.length < 3) {
        alert('Nama minimal 3 karakter.');
        return false;
    }

    if (!email.includes('@')) {
        alert('Format email belum valid.');
        return false;
    }

    if (kelas === '' || jurusan === '') {
        alert('Kelas dan jurusan wajib diisi.');
        return false;
    }

    return true;
}

// Filter data cepat di index
function filterData() {
    const keyword = document.getElementById('cari').value.toLowerCase();
    const rows = document.querySelectorAll('.baris-siswa');

    rows.forEach(function (row) {
        const text = row.textContent.toLowerCase();
        row.style.display = text.includes(keyword) ? '' : 'none';
    });
}

H. Checklist Penyempurnaan CRUD

  • Header tampil rapi di semua halaman CRUD.
  • style.css terhubung dan mengubah tampilan tabel serta form.
  • script.js aktif untuk validasi form dan konfirmasi hapus.
  • Halaman Beranda dan Kontak sudah bisa diakses dari menu.
  • Tombol Simpan, Kembali, Reset, Edit, Hapus berjalan sesuai fungsi.
  • Pencarian cepat di halaman index berjalan.
  • Edit dan update sudah berhasil tanpa error.

I. Latihan Praktik (4 Soal)

  1. Latihan 1: Buat file header.php lalu pasang di beranda.php, index.php, tambah.php, dan kontak.php.
  2. Latihan 2: Buat halaman beranda.php dan kontak.php yang terhubung lewat menu header.
  3. Latihan 3: Perbarui tambah.php dengan tombol Simpan, Kembali, dan Reset.
  4. Latihan 4: Pastikan seluruh navigasi dan CRUD berjalan, termasuk tombol Edit/Hapus dan filter data.
