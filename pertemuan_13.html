Pertemuan 13: Ujian 1 - Website Pribadi

Siswa membangun website pribadi multi-halaman yang rapi, responsif, dan memiliki interaksi JavaScript sederhana.

A. Tujuan Ujian

  • Siswa mampu membuat website pribadi dengan file yang dipisah sesuai fungsi.
  • Siswa mampu menerapkan HTML semantik, CSS terstruktur, dan JavaScript dasar.
  • Siswa mampu menampilkan data diri, foto profil, dan kontak sosial media dengan rapi.

B. Requirement Wajib

  1. Website wajib dipisah menjadi beberapa file HTML, tidak boleh digabung menjadi satu file.
  2. Struktur minimal terdiri dari index.html, tentang.html, dan kontak.html.
  3. Wajib ada file style.css untuk semua halaman.
  4. Wajib ada file script.js untuk interaksi dasar.
  5. index.html berisi halaman beranda / home.
  6. tentang.html berisi data diri siswa dan foto profil.
  7. kontak.html berisi sosial media siswa.
  8. Tampilan nyaman di desktop dan mobile.
  9. Minimal fitur JavaScript, contoh:
    • Tombol scroll ke atas.
  10. Konten memakai identitas diri asli siswa (nama, kelas, kontak).

C. Struktur Folder yang Disarankan

website_pribadi/
├── index.html
├── tentang.html
├── kontak.html
├── style.css
├── script.js
└── assets/
    └── foto-profil.jpg

Catatan: hindari mencampur semua kode dalam satu file agar mudah dinilai. Setiap halaman harus tetap bisa dibuka sendiri dan tetap memakai CSS serta JavaScript yang sama.

D. Kerangka HTML Minimum

Karena file harus dipisah, tiap halaman memakai struktur HTML yang saling terhubung lewat navigasi.

<header>...navigasi ke index.html, tentang.html, kontak.html...</header>
<main>
    <!-- index.html: beranda -->
    <!-- tentang.html: data diri siswa + foto profil -->
    <!-- kontak.html: sosial media siswa -->
</main>
<footer>...copyright...</footer>

Contoh file yang dinilai tetap boleh punya elemen tambahan, seperti tombol tema atau form validasi, selama struktur dasarnya tetap terpisah.

E. Contoh Isi Tiap Halaman

  • index.html: beranda, sapaan singkat, ringkasan profil, dan navigasi ke halaman lain.
  • tentang.html: biodata siswa, foto profil, kelas, hobi, dan deskripsi singkat.
  • kontak.html: daftar sosial media siswa, email, WhatsApp, atau tautan portofolio.
  • style.css: warna, layout, navbar, kartu profil, dan responsif mobile.
  • script.js: toggle tema, validasi form, dan scroll ke atas.
<!-- index.html -->
<a href="index.html">Beranda</a>
<a href="tentang.html">Tentang</a>
<a href="kontak.html">Kontak</a>

<!-- tentang.html -->
<img src="assets/foto-profil.jpg" alt="Foto Profil">
<p>Nama: ...</p>

<!-- kontak.html -->
<a href="https://instagram.com/...">Instagram</a>

F. Jadwal Pelaksanaan Ujian

  1. 13.00 - 15.00: waktu pengerjaan utama project.
  2. 15.30 - 17.00: sesi review, revisi, dan perbaikan kode.

Alokasi kerja saat 13.00 - 15.00: struktur halaman, styling, fitur JavaScript, isi konten, dan uji fungsi dasar.

Alokasi kerja saat 15.30 - 17.00: perbaikan bug, merapikan kode, memastikan responsive, lalu persiapan demo.

G. Rubrik Penilaian

TOTAL 100 POIN
  • Struktur konten dan kelengkapan section (25 poin)
  • Desain visual dan responsive behavior (25 poin)
  • Implementasi JavaScript (20 poin)
  • Kerapian kode dan pemisahan file (15 poin)
  • Kesesuaian konten dengan profil siswa (15 poin)

H. Checklist Sebelum Submit

  • Semua menu di header bisa diklik dan menuju halaman yang benar.
  • Halaman tetap terbaca dengan baik di layar mobile.
  • Fitur JavaScript berjalan tanpa error di console browser.
  • Nama file rapi, tidak ada spasi acak.
  • Tidak ada link atau gambar yang rusak.

I. Format Pengumpulan

  1. Kumpulkan satu folder project bernama: ujian13_nama_kelas.
  2. Pastikan file utama bernama index.html.
  3. Demo singkat 2-3 menit ke pengajar.
