Fase Ujian Output

Pertemuan 13: Ujian 1 - Website Pribadi

Siswa membangun website portofolio pribadi yang rapi, responsif, dan memiliki interaksi JavaScript sederhana.

A. Tujuan Ujian

  • Siswa mampu membuat website pribadi dengan struktur halaman yang jelas.
  • Siswa mampu menerapkan HTML semantik, CSS terstruktur, dan JavaScript dasar.
  • Siswa mampu mempresentasikan hasil project dengan percaya diri.

B. Requirement Wajib

  1. Memiliki section: Beranda, Tentang, Skill, Proyek, dan Kontak.
  2. Memiliki header navigasi yang bisa mengarah ke tiap section.
  3. Tampilan nyaman di desktop dan mobile.
  4. Minimal 2 fitur JavaScript, contoh:
    • Tombol scroll ke atas.
    • Validasi form kontak.
    • Toggle tema terang/gelap.
  5. Memiliki identitas diri asli siswa (nama, kelas, kontak).

C. Struktur Folder yang Disarankan

website_pribadi/
├── index.html
├── style.css
├── script.js
└── assets/
    ├── img-profil.jpg
    └── img-project-1.jpg

Catatan: hindari mencampur semua kode dalam satu file agar mudah dinilai.

D. Kerangka HTML Minimum

<header>...navigasi...</header>
<main>
    <section id="beranda">...</section>
    <section id="tentang">...</section>
    <section id="skill">...</section>
    <section id="proyek">...</section>
    <section id="kontak">...form kontak...</section>
</main>
<footer>...copyright...</footer>

E. Jadwal Pelaksanaan Ujian

  1. 13.00 - 15.00: waktu pengerjaan utama project.
  2. 15.30 - 17.00: sesi review, revisi, dan perbaikan kode.

Alokasi kerja saat 13.00 - 15.00: struktur halaman, styling, fitur JavaScript, isi konten, dan uji fungsi dasar.

Alokasi kerja saat 15.30 - 17.00: perbaikan bug, merapikan kode, memastikan responsive, lalu persiapan demo.

F. Rubrik Penilaian

TOTAL 100 POIN
  • Struktur konten dan kelengkapan section (25 poin)
  • Desain visual dan responsive behavior (25 poin)
  • Implementasi JavaScript (20 poin)
  • Kerapian kode dan pemisahan file (15 poin)
  • Kesesuaian konten dengan profil siswa (15 poin)

G. Checklist Sebelum Submit

  • Semua menu di header bisa diklik dan menuju section benar.
  • Halaman tetap terbaca dengan baik di layar mobile.
  • Fitur JavaScript berjalan tanpa error di console browser.
  • Nama file rapi, tidak ada spasi acak.
  • Tidak ada link atau gambar yang rusak.

H. Format Pengumpulan

  1. Kumpulkan satu folder project bernama: ujian13_nama_kelas.
  2. Pastikan file utama bernama index.html.
  3. Demo singkat 2-3 menit ke pengajar.
