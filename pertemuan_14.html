Pertemuan 14: Ujian 2 - Pendataan Siswa Sederhana (Tanpa Database)

Siswa membuat aplikasi pendataan siswa tanpa MySQL, fokus pada logika proses data, validasi, dan tampilan yang tetap rapi.

A. Tujuan Ujian

  • Siswa memahami alur CRUD sederhana tanpa database.
  • Siswa bisa menyimpan data menggunakan session, JSON, atau file.
  • Siswa tetap menerapkan UI terstruktur dengan header, form, tabel, dan tombol aksi.

B. Requirement Wajib

  1. Form tambah data siswa minimal: nama, kelas, jurusan, email.
  2. Data yang ditambah langsung muncul pada tabel daftar siswa.
  3. Ada tombol hapus data.
  4. Ada validasi input minimal nama tidak boleh kosong.
  5. Header menu tersedia: Beranda, Data Siswa, Kontak.
  6. Terdapat tombol aksi jelas: Simpan, Kembali, Hapus.

C. Opsi Teknik Penyimpanan

  • Session: cepat untuk simulasi di kelas.
  • JSON file: cocok untuk melihat data tersimpan antar reload.
  • Text file (CSV): alternatif paling sederhana.

Rekomendasi kelas: gunakan session untuk waktu ujian yang terbatas.

D. Struktur File yang Disarankan

pendataan_siswa_sederhana/
├── index.php
├── tambah.php
├── hapus.php
├── kontak.php
├── header.php
├── style.css
└── script.js

E. Jadwal Pelaksanaan Ujian

  1. 13.00 - 15.00: waktu pengerjaan utama project.
  2. 15.30 - 17.00: sesi review, revisi, dan perbaikan kode.

Alokasi kerja saat 13.00 - 15.00: buat struktur file, form tambah, tampil data, tombol hapus, dan validasi dasar.

Alokasi kerja saat 15.30 - 17.00: cek bug, rapikan UI, uji tombol aksi, dan pastikan alur demo lancar.

F. Contoh Alur Session Singkat

<?php
session_start();

if (!isset($_SESSION['data_siswa'])) {
    $_SESSION['data_siswa'] = [];
}

if (isset($_POST['simpan'])) {
    $nama = trim($_POST['nama'] ?? '');

    if ($nama !== '') {
        $_SESSION['data_siswa'][] = [
            'nama' => $_POST['nama'],
            'kelas' => $_POST['kelas'],
            'jurusan' => $_POST['jurusan'],
            'email' => $_POST['email']
        ];
    }

    header('Location: index.php');
    exit;
}
?>

Catatan: ini hanya contoh inti logika, siswa tetap wajib melengkapi tampilan HTML dan tombol aksi.

G. Rubrik Penilaian

TOTAL 100 POIN
  • Input dan simpan data (30 poin)
  • Tampil data dan aksi hapus (30 poin)
  • Validasi dan handling input kosong (15 poin)
  • Header, navigasi, dan tombol aksi (15 poin)
  • Kerapian kode dan struktur file (10 poin)

H. Checklist Sebelum Dinilai

  • Tombol Simpan menyimpan data dengan benar.
  • Tombol Hapus benar-benar menghapus data yang dipilih.
  • Tombol Kembali mengarah ke halaman yang sesuai.
  • Header tampil di semua halaman utama.
  • Tidak ada warning/error fatal saat demo.

I. Format Pengumpulan

  1. Nama folder: ujian14_nama_kelas.
  2. File utama: index.php.
  3. Demo 3 menit: tambah data, lihat data, hapus data.
