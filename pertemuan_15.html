Pertemuan 15: Ujian 3 - Sistem Informasi Sekolah CRUD + Presentasi

Ujian puncak: siswa mengintegrasikan front-end, PHP, dan database MySQL dalam satu project CRUD yang stabil lalu mempresentasikannya.

A. Tujuan Ujian Akhir

  • Siswa mampu membangun aplikasi CRUD berbasis database secara utuh.
  • Siswa mampu menunjukkan alur data dari form ke database dan kembali ke tampilan.
  • Siswa mampu menjelaskan arsitektur file project saat presentasi.

B. Requirement Teknis Wajib

  1. Project terkoneksi ke database MySQL melalui config.php.
  2. Fitur Create, Read, Update, Delete berjalan semua.
  3. Header menu tersedia minimal: Beranda, Data Siswa, Tambah Data, Kontak.
  4. Form tambah dan edit memiliki validasi dasar.
  5. Ada konfirmasi sebelum aksi hapus.
  6. Tampilan sudah memakai CSS terpisah dan JavaScript terpisah.

C. Struktur File yang Disarankan

sistem_informasi_sekolah/
├── config.php
├── header.php
├── beranda.php
├── index.php
├── tambah.php
├── edit.php
├── update.php
├── hapus.php
├── kontak.php
├── style.css
└── script.js

D. Checklist Fungsional Sebelum Presentasi

  • Koneksi database berhasil tanpa warning.
  • Tambah data masuk ke tabel database.
  • Data tampil di halaman index.
  • Edit data mengubah isi database dengan benar.
  • Hapus data berhasil dan ada konfirmasi popup.
  • Menu header berpindah halaman tanpa link rusak.

E. Jadwal Pelaksanaan Ujian

  1. 13.00 - 15.00: waktu pengerjaan utama project CRUD.
  2. 15.30 - 17.00: sesi review, revisi, dan perbaikan kode sebelum presentasi.

Alokasi kerja saat 13.00 - 15.00: finalisasi fitur CRUD, validasi, konfirmasi hapus, dan pengecekan koneksi database.

Alokasi kerja saat 15.30 - 17.00: debug akhir, rapikan tampilan, latihan presentasi, dan uji demo end-to-end.

F. Alur Presentasi Siswa (10 Menit)

  1. 1 menit: perkenalan project dan tujuan aplikasi.
  2. 2 menit: jelaskan struktur file (config, header, halaman CRUD).
  3. 4 menit: demo alur CRUD lengkap (tambah, tampil, edit, hapus).
  4. 2 menit: jelaskan CSS/JS yang dipakai (validasi, konfirmasi, navigasi).
  5. 1 menit: penutup dan rencana pengembangan.

G. Format Tanya Jawab Penguji

  • Tunjukkan letak file koneksi database.
  • Jelaskan perbedaan halaman index, tambah, edit, update, hapus.
  • Tunjukkan bagian validasi input pada form.
  • Jelaskan kenapa konfirmasi hapus penting.

H. Rubrik Penilaian Akhir

TOTAL 100 POIN
  • CRUD berfungsi end-to-end (40 poin)
  • Struktur kode dan file project (15 poin)
  • Tampilan UI dan konsistensi halaman (15 poin)
  • Validasi dan error handling dasar (10 poin)
  • Kualitas presentasi dan pemahaman konsep (20 poin)
