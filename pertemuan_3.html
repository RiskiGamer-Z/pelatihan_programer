A. Tiga Cara Menulis CSS & Selector

Sesuai standar materi industri, terdapat 3 pendekatan untuk menyisipkan CSS ke HTML beserta cara memanggil elemennya (Selector):

  • 1. External Style Sheet (Pendekatan Skalabilitas): Kode CSS dipisah pada file .css sendiri lalu di-import lewat tag <link>. (Pendekatan terbaik untuk proyek besar).
  • 2. Internal Style Sheet (Pendekatan Cepat/Isolasi): Diletakkan di dalam tag <head> HTML menggunakan tag <style>.
  • 3. Inline Style Sheet (Pendekatan Darurat): Diletakkan langsung di dalam tag HTML target, misalnya: <h1 style="color:red;">.

💻 Contoh Kode 1: Internal CSS & Selector Element

<head>
  <style>
    /* CSS Internal: Mengubah semua tag <p> menjadi biru */
    p { color: blue; font-family: Arial; }
  </style>
</head>
<body>
  <p>Saya berwarna biru!</p>
</body>
🔍 Bedah Kode Pertama:
  • <style>: Wadah/Kamar murni untuk menampung seluruh kode spesifikasi CSS agar tidak tersentuh atau bercampur baur berantakan dengan struktur dasar HTML.
  • p { ... }: Memanggil dan memilih seluruh pasukan tag <p> (paragraf) yang ada di halaman ini sekaligus. Efisiensi luar biasa.

💻 Contoh Kode 2: Selector Class vs ID

<style>
  /* Class digunakan massal (Tanda Titik) */
  .tombol { background-color: yellow; padding: 10px; }
  
  /* ID bersifat unik/kritis (Tanda Pagar) */
  #logo-utama { color: red; font-weight: bold; }
</style>

<button class="tombol">Klik 1</button>
<button class="tombol">Klik 2</button>
<div id="logo-utama">MY BRAND</div>
🔍 Bedah Karakter Selector (+ Pengenalan "Div"):
  • <div>: Sangat Penting! (Pendek dari Division). Tag HTML transparan alias tanpa bentuk/gaya bawaan apapun. Ibarat laci laci kosong di lemari Anda. Tugas khususnya HANYA SEBAGAI wadah kerangka / blok pembungkus tata letak (Layout).
  • .tombol: Penggunaan titik (.) dalam CSS menandakan pembuatan Class. Stempel yang didesain untuk ditempelkan secara massal (berkali-kali).
  • #logo-utama: Penggunaan pagar (#) adalah ID. Desain kritis eksklusif satu-satunya di halaman itu dan pantang diduplikat.

B. Box Model: Margin, Border, & Padding

Dalam web-design modern, setiap elemen wajib diperlakukan persis layaknya "Kotak" (Box Model).

  • Padding (Ruang Dalam): Jarak antara isi teks/konten menuju batas garis kotaknya (Border).
  • Border: Garis batas fisik (Tepi) dari kotak tersebut.
  • Margin (Ruang Luar): Jarak saling dorong / jarak spasi ANTAR elemen satu dengan elemen lainnya.

💻 Contoh Kode 1: Senggolan Margin vs Padding Luas

<style>
  .kotak-pesan {
    background-color: lightgray;
    padding: 20px; /* Jarak teks ke sisi/tepi box jadi luas */
    margin: 30px;  /* Box ini mendorong semua elemen di sekelilingnya sejauh 30px */
    border: 2px solid black; /* Garis pembatas kotak */
  }
</style>
<div class="kotak-pesan">Bebas sesak! Saya punya ruang napas dalam kotak ini!</div>
🔍 Bedah Kode Jaga Jarak:
  • padding: 20px;: Mengetuk palu ruang gerak di dalam! Teks tulisan yang bersandar di pinggir TEMBOK Kotak kini dijauhkan renggang 20 pixel (Mencegah nuansa sesak/sempit).
  • margin: 30px;: Menguasai radius wilayah sekeliling elemen! Bagian luaran kotak ini akan otomatis menendang menolak semua bentuk elemen tetangganya yang coba mendekat sejauh 30 pixel.
  • border: 2px solid black;: Membangun pagar fisik fisik sesungguhnya. Berupa garis arsitektur tegas berkelir hitam setebal 2 pixel.
🐛 Potensi Bug Dimensi Box Model: Menambahkan ukuran Padding sering kali "ikut menghitung" dan membuat ukuran lebar/tinggi utuh kotak malah membengkak liar melebihi ekspektasi Anda!
🚀 Solusi Clean Code: Suntikkan kode peretas reset * { box-sizing: border-box; } di barisan paling atas CSS Anda, agar Padding mengalah ke dalam dan tidak merusak susunan ukuran pasti kotak.

C. Meratakan Layout Menggunakan Flexbox Sempurna

Flexbox adalah sihir tata letak modern. Mengakhiri penderitaan susunan elemen yang selalu menumpuk kaku ke bawah, Flexbox membariskan anak-anaknya secara elegan sejajar (secara sentral atau menyebar rata).

💻 Contoh Kode 1: Kekuatan Sentralisasi Vertikal & Horizontal

<style>
  .wadah-tengah {
    display: flex; /* Aktifkan sihir kotak fleksibel pada Induk */
    justify-content: center; /* Rata mendatar sempurna */
    align-items: center; /* Rata vertikal (Tengah secara tinggi) sempurna! */
    height: 150px; /* Induk Butuh tinggi, jika meratakan secara vertikal */
    background-color: #eeddff;
  }
</style>
<div class="wadah-tengah">Saya mengapung tepat persis di tengah kotak ini!</div>
🔍 Bedah Keajaiban Sihir 'Flex':
  • display: flex;: adalah sistem tata letak (layout) yang membuat sebuah elemen (wadah) menjadi fleksibel, sehingga Anda bisa dengan sangat mudah mengatur posisi, penyelarasan, dan jarak elemen-elemen di dalamnya.
  • justify-content: center;: (Titik Berat Horizontal / Mendatar). Memikat dan menyeret gerombolan anak pas di sumbu X Tengah.
  • align-items: center;: (Titik Berat Vertikal / Tegak Lurus). Menarik gerombolan pas presisi ke di tengah sumbu Y. (Catatan: Hukum align-items ini hampa khasiat bila tinggi induk/wadah tidak didefinisikan pakai height!)

💻 Contoh Kode 2: Navbar Membelah (Kiri & Kanan Terpisah)

<style>
  .navbar-ajaib {
    display: flex;
    justify-content: space-between; /* Induk melempar satu sisi mentok ke kiri, satu ke kanan */
    padding: 15px;
    background-color: #333; color: white;
  }
</style>
<div class="navbar-ajaib">
  <h3>LOGO STARTUP</h3>
  <div>Menu | Fitur Utama | Login</div>
</div>
🔍 Bedah Taktik Navbar Belah Lautan (Space-Between):

Mungkin teknik sakti properti CSS yang akan 100% Anda gunakan dalam merakit gerbang atas/Navbar aplikasi: justify-content: space-between;. Menolak untuk merapat bertumpuk, kemampuan pamungkas Flexbox ini membanting elemen ke-1 bersandar menyendiri mentok di pojok ujung Kiri Layar, dan menyeret sisa elemen lainnya berkumpul tertahan di sudut Kanan ujung layar!

D. Mendominasi Visual: File Gambar Lokal & Background CSS

Selain URL web (seperti contoh sebelumnya), Anda mutlak wajib menggunakan file gambar asli dari folder Anda sendiri jika sedang memajang Logo atau Icon perusahaan agar tidak tergantung koneksi WiFi saat *offline*.

💻 Contoh Kode 1: Tag Elemen Gambar File Lokal (Local Assets)

<!-- Tag img tidak punya penutup. Jika file gambar beda folder, tulis petunjuk arahnya (misal: bg/logo.png)! -->
<img src="logo_asli.png" alt="Logo Startup" class="foto-bundar">

<style>
  .foto-bundar {
    width: 150px; /* Kotak paksa agar proporsional */
    height: 150px;
    object-fit: contain; /* Contain: mencegah gambar terpotong seperti logo! Pakai Cover jika ingin full edge */
    border-radius: 50%; /* Memutar logo kotak menjadi bulat sempurna */
    border: 3px solid white;
  }
</style>
🔍 Bedah Anatomi Elemen Gambar & Pembulatan Sudut:
  • <img src="..." alt="...">: alt ini adalah teks pengganti yang muncul jika gambar gagal dimuat.
  • object-fit: cover;: Mencegah gambar berontak karena dipaksa melar gepeng *stretching*. Gambar dipastikan memotong / sembunyi secara elegan ke belakang bingkai kotak demi mengamankan harga diri rasionya *aspect-ratio*.
  • border-radius: 50%;: Pemahat ujung kotak paling manjur sejauh ini! Melelehkan keempat pinggiran kotak selebar 50% hingga mekar menjadi pilar Bulat Lingkaran Total Sejati.

💻 Contoh Kode 2: Latar Belakang Epik (CSS Background)

<style>
  .header-landing-page {
    width: 100%;
    height: 400px;
    
    /* Memanggil lukisan grafis dan meriasnya merata di dinding latar! */
    background-image: url('https://picsum.photos/1000/400');
    background-size: cover; /* Gambar me-resize proporsional mentok hingga sudut kotak */
    background-position: center; /* Titik fokus gambar adalah centernya */
    color: white; 
    
    /* Kombinasi Flexbox memposisikan teks Judul ke tengah */
    display: flex; justify-content: center; align-items: center; 
  }
</style>

<div class="header-landing-page">
  <h1 style="text-shadow: 2px 2px 5px black;">Selamat Datang di Portofolio Megahku!</h1>
</div>
🔍 Bedah Kode Latar Belakang Dinding Mural:
  • background-image: url('...');: Memanggil dan menyerap grafis untuk mendasar / menempel di lantai kotak (menjadi latar) bukan tampil sebagai obyek solid murni, sehingga di atasnya masih bisa dijejali elemen ketikan tulisan dengan merdeka terbang!
  • background-size: cover;: Memaksa gambar pemandangan untuk ditarik atau dipres demi memenuhi seluruh ruang kosong di dinding batas elemen sekeras apapun takdir kotak <div> membatasi tinggi dan lebarnya.
  • text-shadow: ...: Trik muslihat UI Profesional! Menyisipkan gumpalan bayangan redup hitam pada huruf teks cemerlang, memastikan tulisan tetap memancarkan keterbacaan tinggi saat latar gambarnya sedang silau warna keputihan terang sekalipun.
💡 Tip URL / Path Asli (Lokal):
Jika file `html` Anda sejajar berdampingan dengan gambar foto, cukup panggil nama filenya: src="kucing.jpg". Namun, sangat dianjurkan menyimpan kumpulan gambar di folder mandiri untuk Web Pro! Jadi formatnya: src="images/kucing.jpg"
🍌 Nano Banana Tips: Sebagai UI Designer sejati, tidak ada istilah "Ngoding Macet karena Nunggu Foto!". Gunakan layanan Placeholder keren seperti https://picsum.photos/ sebagai pemancing visual *(dummy file)* sementara saat file gambar logo asli masih direvisi!
