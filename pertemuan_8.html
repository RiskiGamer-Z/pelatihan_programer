  • Menerapkan form processing dalam aplikasi sederhana

B. Konsep Dasar Fungsi PHP

Fungsi adalah blok kode yang dirancang untuk melakukan tugas tertentu. Fungsi dapat dipanggil berkali-kali dalam program untuk mengurangi pengulangan kode (DRY - Don't Repeat Yourself).

Manfaat menggunakan fungsi:

  • Mengurangi duplikasi kode
  • Membuat kode lebih terorganisir dan mudah dibaca
  • Mempermudah debugging (perbaikan kesalahan)
  • Memungkinkan reusable code (kode yang dapat digunakan kembali)
  • Mempercepat development (pengembangan)

Struktur Dasar Fungsi:

  • function - keyword untuk mendefinisikan fungsi
  • namaFungsi - nama yang diberikan pada fungsi
  • parameter - input yang diterima fungsi (opsional)
  • return - nilai yang dikembalikan oleh fungsi (opsional)

C. Jenis-Jenis Fungsi PHP

1. Fungsi Tanpa Parameter dan Tanpa Return

Fungsi paling sederhana yang hanya menjalankan perintah tanpa menerima input atau mengembalikan nilai.

<?php
function salam() {
    echo "Selamat datang di Pelatihan Programmer!";
}

// Memanggil fungsi
salam(); // Output: Selamat datang di Pelatihan Programmer!
?>

2. Fungsi Dengan Parameter

Fungsi yang menerima input (parameter) untuk memproses data sesuai kebutuhan.

<?php
function sapaNama($nama, $usia) {
    echo "Halo, nama saya " . $nama;
    echo " dan umur saya " . $usia . " tahun";
}

// Memanggil fungsi dengan parameter
sapaNama("Budi", 25);      // Output: Halo, nama saya Budi dan umur saya 25 tahun
sapaNama("Siti", 22);      // Output: Halo, nama saya Siti dan umur saya 22 tahun
?>

3. Fungsi Dengan Return Value

Fungsi yang mengembalikan nilai hasil proses untuk digunakan di tempat lain dalam program.

<?php
function hitungJumlah($a, $b) {
    $hasil = $a + $b;
    return $hasil;
}

// Memanggil fungsi dan menyimpan return value
$total = hitungJumlah(10, 15);
echo "Total: " . $total;  // Output: Total: 25

// Atau langsung menampilkan
echo hitungJumlah(5, 3);  // Output: 8
?>

4. Fungsi Dengan Parameter dan Return

Kombinasi parameter dan return value untuk membuat fungsi yang fleksibel dan powerful.

<?php
function hitungLuasPersegiPanjang($panjang, $lebar) {
    $luas = $panjang * $lebar;
    return $luas;
}

function hitungKelilingPersegiPanjang($panjang, $lebar) {
    $keliling = 2 * ($panjang + $lebar);
    return $keliling;
}

// Menggunakan fungsi
$luasPersegiPanjang = hitungLuasPersegiPanjang(10, 5);    // 50
$kelilingPersegiPanjang = hitungKelilingPersegiPanjang(10, 5); // 30

echo "Luas: " . $luasPersegiPanjang . " cm²\n";
echo "Keliling: " . $kelilingPersegiPanjang . " cm";
?>

D. Scope Variabel dalam Fungsi

Scope menentukan bagian mana dari program yang dapat mengakses suatu variabel.

1. Local Scope

Variabel yang dideklarasikan di dalam fungsi hanya dapat diakses di dalam fungsi tersebut.

<?php
function testLocal() {
    $nama = "Budi";
    echo $nama;  // Output: Budi (bisa diakses di dalam fungsi)
}

testLocal();
echo $nama;  // ERROR! Undefined variable. $nama tidak bisa diakses di sini
?>

2. Global Scope

Variabel yang dideklarasikan di luar fungsi memiliki global scope, tetapi untuk mengaksesnya dari dalam fungsi, harus menggunakan keyword global.

<?php
$nama = "Siti";  // Global scope

function testGlobal() {
    global $nama;  // Deklarasi variabel global di dalam fungsi
    echo $nama;    // Output: Siti (sekarang bisa diakses)
}

testGlobal();
echo $nama;  // Output: Siti (bisa diakses dari luar fungsi)
?>

3. Super Global Variables

Variabel super global adalah variabel yang sudah dideklarasikan oleh PHP dan dapat diakses di mana saja tanpa perlu keyword global. Contoh: $_GET, $_POST, $_SERVER, $_FILES, dll.

<?php
function testSuperglobal() {
    // Super global variables bisa diakses tanpa keyword global
    echo $_SERVER['PHP_SELF'];   // Nama file PHP yang sedang dijalankan
    echo $_SERVER['HTTP_HOST'];  // Nama host
}

testSuperglobal();
?>

E. Form Processing: Metode GET dan POST

Perbedaan GET dan POST

Aspek GET POST
Visibilitas Data Terlihat di URL Disembunyikan (di header HTTP)
Keamanan Kurang aman (data bisa dilihat) Lebih aman (data tidak terlihat)
Ukuran Data Terbatas (~2000 karakter) Tidak terbatas
Kegunaan Pencarian, filter, parameter halaman Login, upload file, data sensitif
Caching Bisa di-cache oleh browser Tidak di-cache

F. Fungsi-Fungsi Penting untuk Form Processing

1. isset() - Mengecek Apakah Variabel Ada

Digunakan untuk memastikan data dari form sudah dikirim sebelum diproses.

<?php
if (isset($_GET['nama'])) {
    echo "Data nama diterima: " . $_GET['nama'];
} else {
    echo "Data nama belum dikirim";
}
?>

2. htmlspecialchars() - Mengamankan Data Output

Mengkonversi karakter khusus menjadi HTML entities untuk mencegah XSS attack (keamanan).

<?php
// Tanpa htmlspecialchars() - BERBAHAYA
$nama = "<script>alert('Hacked!')</script>";
echo $nama;  // Script akan dijalankan!

// Dengan htmlspecialchars() - AMAN
echo htmlspecialchars($nama);  // Script ditampilkan sebagai text biasa
?>

3. empty() - Mengecek Apakah Variabel Kosong

Mengecek apakah variabel kosong, belum diset, atau bernilai 0/false.

<?php
$nama = "";

if (empty($nama)) {
    echo "Nama tidak boleh kosong!";
} else {
    echo "Nama: " . $nama;
}
?>

4. $_SERVER['REQUEST_METHOD'] - Mengecek Metode Form

Mengecek apakah form disubmit dengan GET atau POST.

<?php
if ($_SERVER['REQUEST_METHOD'] === 'POST') {
    // Proses data POST
    echo "Form disubmit dengan POST";
} elseif ($_SERVER['REQUEST_METHOD'] === 'GET') {
    // Proses data GET
    echo "Form disubmit dengan GET";
}
?>

G. Contoh Form Processing Lengkap

Contoh 1: Form Pendaftaran Siswa (POST)

<?php
if ($_SERVER['REQUEST_METHOD'] === 'POST') {
    $nama = htmlspecialchars($_POST['nama'] ?? '');
    $email = htmlspecialchars($_POST['email'] ?? '');
    
    if (!empty($nama) && !empty($email)) {
        echo "Pendaftaran berhasil! Nama Anda: " . $nama;
    } else {
        echo "Semua field harus diisi!";
    }
}
?>

<form method="POST" action="">
    <input type="text" name="nama" placeholder="Nama" required>
    <input type="email" name="email" placeholder="Email" required>
    <button type="submit">Daftar</button>
</form>

Contoh 2: Kalkulator Sederhana (GET)

<?php
$hasil = "";

if (isset($_GET['angka1']) && isset($_GET['angka2']) && isset($_GET['operasi'])) {
    $angka1 = floatval($_GET['angka1']);
    $angka2 = floatval($_GET['angka2']);
    $operasi = $_GET['operasi'];
    
    if ($operasi === 'tambah') $hasil = $angka1 + $angka2;
    elseif ($operasi === 'kurang') $hasil = $angka1 - $angka2;
    elseif ($operasi === 'kali') $hasil = $angka1 * $angka2;
    elseif ($operasi === 'bagi') $hasil = ($angka2 != 0) ? $angka1 / $angka2 : "Error";
}
?>

<form method="GET" action="">
    <input type="number" name="angka1" placeholder="Angka 1" required>
    <select name="operasi" required>
        <option value="tambah">Tambah (+)</option>
        <option value="kurang">Kurang (-)</option>
        <option value="kali">Kali (*)</option>
        <option value="bagi">Bagi (/)</option>
    </select>
    <input type="number" name="angka2" placeholder="Angka 2" required>
    <button type="submit">Hitung</button>
</form>

I. Kesimpulan dan Tips

Poin Kunci:

  • Fungsi mengurangi duplikasi dan membuat code terstruktur
  • Parameter memungkinkan input yang berbeda-beda
  • Return value mengembalikan hasil untuk digunakan di tempat lain
  • Scope variabel penting untuk akses variabel yang benar
  • GET untuk data non-sensitif (terlihat di URL)
  • POST untuk data sensitif (tersembunyi)
  • Validasi dan sanitasi adalah keharusan untuk keamanan

Tips Pengembangan:

  • Buat nama fungsi yang deskriptif dan jelas
  • Gunakan comments untuk logika yang kompleks
  • Test dengan berbagai macam input
  • Selalu gunakan htmlspecialchars() pada output
  • Validasi input dari user sebelum pemrosesan
